# Customer copy map

The polished study uses direct customer language. Internal class names and synthetic fixture keys are not customer-facing and are outside this copy map.

| Previous text | Final text | Reason | Surface |
|---|---|---|---|
| `Operational access` | Removed | Internal-sounding label duplicated the page purpose. | Launcher |
| `App launcher` | `Your product` | Describes the one-product reality without marketplace or technical launcher language. | Launcher |
| `Open the product available to your current workspace.` | `Open the product available to your workspace.` | Keeps the useful direction while removing an unnecessary qualifier. | Launcher |
| `Workspace product` | `Quotations` | Replaces system classification language with the customer's task. | Launcher |
| `Create accurate, branded quotations from your approved products, pricing rules and commercial terms.` | `Create professional quotations using your approved workspace.` | Shorter, truthful, and aligned with the requested customer wording. | Launcher |
| `Workspace: Swooshz Studio` inside the product unit | Removed | Workspace is already present in the shell; repeating the named context made the unit feel procedural. | Launcher status |
| `Access: Active` and `Session: Ready` | `Product status: Ready to launch` | Replaces internal readiness fields with one clear customer state. | Launcher status |
| `Launch quote generator` | `Launch product` | Names the action plainly and keeps the full product name nearby. | Launcher action |
| `Opening quote workspace...` | `Opening product...` | Removes internal destination terminology. | Launcher loading |
| `Launch could not complete. Your session is safe. Check your connection and try again.` | `We could not open the product. Try again.` | Direct recovery copy without architecture or unsupported reassurance. | Launcher error |
| `Retry launch` | `Retry` | The surrounding error already identifies the action. | Launcher error action |
| `Product unavailable` | `Launch product` (disabled) | Keeps the action label stable while status and explanation communicate why it cannot run. | Launcher unavailable action |
| `Product access is unavailable. Ask a workspace administrator to check the current entitlement.` | `Product access unavailable. Your workspace does not currently have access to this product.` | Removes internal data-model language and avoids assigning an unsupported support process. | Launcher unavailable status |
| `Secure workspace handoff. Opens the quote workspace with your current access and workspace context.` | `Ready when you are. Your workspace and role stay the same when the product opens.` | Replaces architecture language with a concise description of the customer experience. | Launcher supporting line |
| `Protected` | `Protected owner` | Makes the reason for the non-action state explicit. | Admin status |
| `Workspace owner is protected from removal.` | `The workspace owner cannot be removed.` | Uses direct customer language. | Admin supporting text |
| `Entitlement` | `Product access` | Uses the approved administration section terminology. | Admin status |
| `Disable access` | `Disable product access` | Keeps the action explicit and distinct from member/workspace access. | Admin action |
| `Members with workspace access can create and manage quotations.` | `Members with access can create professional quotations for this workspace.` | Clarifies the scope and aligns with the final product description. | Admin Product access |
| `Disabling access prevents new launches but does not change member records.` | `Disabling product access prevents new launches and does not change member records.` | Names the affected access type precisely. | Admin supporting text |
| `Review recent bounded administrative actions for this workspace.` | `Review recent administrative changes for this workspace.` | Removes prototype/testing language. | Admin Audit activity |
| `Workspace member` | `Members` | Matches the section name and removes a system label. | Modal |
| `Add an existing account to Swooshz Studio. This study does not send email.` | `Add an existing Swooshz account to this workspace.` | Removes prototype disclosure from customer UI and does not invent invitation delivery. | Modal |
| `Member could not be added. Check the account and try again.` | `We could not add this member. Check the email address and try again.` | Gives a direct recovery step without technical detail. | Modal error |
| `Members will be unable to start new quote sessions until access is enabled again.` | `Members will be unable to launch this product until access is enabled again.` | Removes internal session terminology. | Confirmation |
| `Confirm role` | `Change role` | Matches the action the customer initiated. | Confirmation action |

## Final customer-visible status and action vocabulary

- Active
- Disabled
- Pending
- Protected owner
- Ready to launch
- Product access unavailable
- Add member
- Change role
- Disable member
- Reactivate member
- Remove member
- Revoke approval
- Enable product access
- Disable product access
- Launch product
- Retry
