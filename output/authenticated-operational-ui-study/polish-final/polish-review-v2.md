# Polish review - Version 2

Evidence: `evidence/core-v2/`
Measurements: `polish-v2-metrics.json`

## What still felt generic in version 1

- The first quotation symbol had the silhouette of a generic file even though its subject was correct.
- The active administration item used a familiar solid highlight with little Swooshz-specific edge logic.
- The table header applied one uniform tint across every column, so member identity did not lead strongly enough.
- The 96px action column forced `Protected owner` onto two lines.
- The launcher introduction repeated the product/task relationship, and its closing band did not yet echo the product/readiness column structure.

## What changed

- Redrew the study-only inline SVG as a structured quote sheet: folded document edge, three line-item rules, a calculation column, and a stronger total rule.
- Increased the desktop symbol from 48px to 52px while preserving its 34px mobile size and subordinate visual role.
- Added a navy close to the lower part of the launcher leading edge and aligned a restrained two-material footer treatment with the desktop product/readiness split. Mobile deliberately returns to one continuous material.
- Replaced the active-nav block with a three-pixel teal leading rule plus the existing bottom rule and pale ground.
- Gave the member identity header a slightly stronger teal-tinted material, added a quiet identity-column rule, and applied the same split logic only to hover/focus states.
- Widened the desktop action column to 124px so `Protected owner`, `Reactivate`, and `Manage` align cleanly.
- Tightened the launcher introduction to `Open the product available to your workspace.` and rewrote the closing line in plain language.

## Distinctiveness and operational character

Version 2 is more distinctive because the same edge logic now connects the shell, launcher, active navigation, member identity column, and status/action hierarchy. It remains operational rather than promotional: type is modest, copy is task-focused, content is truthful, and no new destination, widget, product, metric, or illustration was added.

## Remaining weaknesses

- The truthful one-product desktop launcher still has quiet page depth below the product unit. It now reads as intentional calm, but only real future workflow content could change that further.
- The quotation symbol is study-only and must receive brand/product-owner approval before production use.
- The Members surface remains a table on desktop because that is the correct administration pattern; its distinctiveness is intentionally restrained.
- Browser-plugin startup timed out before page access, so rendered validation used local Playwright with installed Chrome. Production browser and assistive-technology coverage remains separate work.
