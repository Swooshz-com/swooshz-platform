# Polish review - Version 1

Evidence: `evidence/core-v1/`
Measurements: `polish-v1-metrics.json`

## Gate result

Version 1 preserves the approved shell, launcher, administration structure, typography scale, responsive transformation, and interaction model. It passes the technical screen gate, but it is not final: the new product symbol still reads too close to a generic document at small size, the Members action column does not comfortably hold `Protected owner`, and the table/navigation treatment remains familiar enough to need one more authored pass.

| Category | Score | Visual basis |
|---|---:|---|
| Brand distinctiveness | 4 | The Swooshz mark, content-aligned teal/navy top rule, warmer page ground, and teal leading edges feel owned. The administration surface still uses a familiar table silhouette. |
| Product identity | 3 | A quotation-document symbol replaces the unrelated Swooshz brand mark inside the product unit, but its line detail collapses into a generic file shape at mobile size. |
| Copy naturalness | 4 | `Your product`, `Ready to launch`, `Launch product`, and the concise product description remove prototype and architecture language. The launcher intro is slightly repetitive, and the footer still leans on `workspace access`. |
| Desktop launcher balance | 4 | The complete launcher object now sits deliberately within the usable viewport and no longer clings to the top. The right status/action panel is cohesive, though the wide flat footer could carry a more precise closing rhythm. |
| Admin refinement | 4 | Stronger member identity, squared avatars, distinct status treatments, a firmer table header, visible hover/focus states, and a closer heading-to-surface relationship add polish. `Protected owner` wraps and the header remains conventional. |
| Typography consistency | 5 | Body and primary row text remain 16px, supporting text remains 14-15px, controls remain 15px, and page titles remain 34px desktop / 28px mobile. |
| Mobile preservation | 5 | At 390px the full product name, description, status, action, workspace, and role are visible without clipping; member rows retain their identity-first transformation and comfortable touch sizes. |
| Generic SaaS risk | 3 | The absence of a sidebar, cards, metrics, and marketplace keeps the overall composition specific, but the active tab block and table header could still belong to another competent admin product. |
| Marketing-page risk | 5 | Modest type, one practical action, no illustration stage, and no promotional content keep the launcher operational. |
| Overall beauty | 4 | V1 is calm, coherent, and clearly improved, but the icon, action-column fit, and administration material hierarchy prevent a fully authored finish. |

## Required v2 refinement

- Rework the inline symbol into a clearer structured quote sheet with line-item and total cues.
- Widen and align the desktop action column so `Protected owner` remains on one line.
- Give the identity column and table header a more deliberate material hierarchy without zebra striping.
- Replace the generic active-tab block with a restrained leading-rule treatment that remains horizontal navigation.
- Tighten launcher intro/footer copy and edge hierarchy without adding content.
- Preserve the current mobile viewport positions, type sizes, and row transformation.
